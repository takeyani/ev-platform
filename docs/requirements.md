# EV充電器施工管理プラットフォーム - 要件定義書

## 1. プロジェクト概要

### 1.1 目的
Terra Charge社のEV充電器設置工事における業務プロセスをデジタル化し、
Google Sheets・Kizuku・Toyokumoに分散している業務情報を一元管理するプラットフォームを構築する。

### 1.2 対象ユーザー
| ロール | システム権限 | 説明 | 主な操作 |
|--------|------------|------|---------|
| admin | 全操作 | システム管理者 | マスタ管理、権限管理、案件削除 |
| terra_case | 全操作（削除以外）| テラ案件担当。お客様窓口・補助金対応 | 案件登録、日程調整、検収承認 |
| terra_const | 閲覧のみ | テラ施工管理。技術的窓口 | 図面確認、施工要領相談、完了報告精査 |
| contractor | 自社案件の閲覧・更新 | 協力会社。施工実施 | 工事進捗報告、書類提出、報告書作成 |
| manufacturer | 自社充電器案件の閲覧 | 充電器メーカー | 仕様書閲覧 |

### 1.3 ビジネスコンテキスト
- **事業**: Terra Charge = EV充電器の設置・運用サービス
- **補助金**: NeV（次世代自動車振興センター）の交付決定に基づく工事が大半
- **施工体制**: Terra Charge → PANDORA（通常業務責任者:西平）→ 協力会社

### 1.4 R8年度 組織体制
**テラ担当者（エリア制）**:
| エリア | 担当 | 拠点 | 管轄 |
|--------|------|------|------|
| 東日本 | 浅井 | 東京本社 | 神奈川県から東エリア全て |
| 中日本 | 関 | 東京本社 | 新潟/富山/石川/福井、静岡/長野/愛知/岐阜/三重 |
| 西日本 | 大八木 | 関西支店 | 滋賀県から西エリア全て |

**PANDORA社内体制（5名）**:
- 代表: 本居社長（テラ役員面談、契約、大型案件）
- 通常業務責任者: 西平（営業、施工管理、電力申請、協力会社選定）
- CAD図面: 松本（リブレ）、安部（タッグオンユー）— 外注
- 施工管理兼施工: 當山（S-Service）— 外注、自社施工可能案件担当

### 1.5 案件種別（EV補助金案件種別）
| 種別 | 対象物件 | 充電器 | 費用負担 |
|------|---------|--------|---------|
| ①基礎充電 | 集合住宅（賃貸・分譲）| 3kW NOEX製のみ | イニシャル一部負担あり |
| ②目的地充電 | 商業施設・宿泊施設等 | 6kW（日東/河村/Zerova）| 全額テラ負担 |
| ③目的地(急速) | 大型商業施設、テラ指定案件 | 50kW（新電元/ニチコン/ダイヘン）| 全額テラ負担 |
| ④目的地(超急速) | ③と同条件＋第一種電気工事士必須 | 超急速 | 案件毎確認 |

## 2. 業務フロー要件

### 2.1 工事全体フロー（11ステップ）
```
1. 交付決定確認 → 2. お客様連絡 → 3. 施工発注 → 4. 日程調整
→ 5. 着工前会議 → 6. 安全書類 → 7. 充電器発送・資材確保
→ 8. 着工Ready確認 → 9. 工事実施 → 10. 完了報告 → 11. 検収・請求
```

### 2.2 各ステップの詳細要件

#### Step 1-2: 交付決定〜お客様連絡
- 補助金団体からの交付決定を確認
- テラ営業 → お客様へ交付決定連絡
- 正式発注前の先行連絡は**禁止**

#### Step 3: 施工発注
- メール（発注書添付）での正式発注
- 発注書には案件ID・案件名・金額・支払条件を記載
- 支払条件: 工事完了報告書承認後、末締・翌月末払い

#### Step 4: 日程調整
- 着工前会議日程と施工日程を調整
- テラ調整 or 協力会社調整（案件毎に確認）
- 進捗シートに全情報を記入

#### Step 5: 着工前会議
- 必ずお客様と実施（現地 / Web / 電話）
- テラ参加必要案件は三者間調整
- **議事録作成必須**（着工前会議後2営業日以内にKizukuに提出）
- 議事録項目:
  - 工事概要・内容説明（工程、可能時間、停電日、主技立会、電力申請）
  - 仮設計画（作業エリアと区画）
  - 車両移動・工事告知・ポスティング
  - 石綿調査（レベル3みなし/レベル1/2）
  - 当日緊急連絡先
  - エレベーター/宅配ボックス/コインパ影響確認

#### Step 6: 安全書類
- **着工1週間前まで**にKizukuへ提出
- 14種類の安全書類（後述）
- 年度毎 + 工事毎の2種類の提出サイクル

#### Step 7: 充電器発送・資材確保
- 工事1ヶ月前までに送付先入力
- 充電器納期: 約2週間
- 着工1週間前までに全資材確保
- QRシール管理（新モデル: 貼付済み / 23年モデル: 現場貼付）
- デバイスID管理（製造番号、デバイスIDシリアル、マイ充電器コード、QR送付日）

#### Step 8: 着工Ready確認
- 全入力項目完了で自動判定 → テラ確認で「着工Ready」確定
- 必須項目: 着工予定日、完工予定日、現場担当者、産廃情報、安全書類

#### Step 9: 工事実施
- Kizuku/電話での連絡
- 毎日: 作業開始連絡、工事写真提出、終了連絡
- 変更発生時: 直ちにテラへ連絡、補助金確認まで工事中断

#### Step 10: 完了報告
- **完工後3営業日以内**に提出
- 必要書類: 工事完了報告書、写真撮影チェックシート、完成図面
- 充電器・分電盤の鍵をお客様に引渡し（3kWは不要）

#### Step 11: 検収・請求
- **毎月20日まで**: 図面提出 + 報告書承認依頼
- **毎月25日まで**: 承認 → 請求対象確定
- **月初2営業日以内**: 請求書PDF送付
- 請求書窓口: seko-invoice@terra-charge.co.jp
- 現調費/図面費と工事費は分けて発行

## 3. データ要件

### 3.1 案件（Project）データモデル
進捗共有シートの全カラムに対応する60+フィールド。

#### 基本情報
| フィールド | 型 | 説明 | 例 |
|-----------|-----|------|-----|
| caseId | string | 案件ID (英字1+数字4桁) | T05010 |
| nevId | string | NevID | 205697 |
| subsidyType | enum | 補助金区分 | R8 Nev |
| name | string | 案件名（補助金の名前） | ○○マンション |
| prefecture | string | 都道府県 | 東京 |
| location | string | 住所 | 東京都港区芝浦3-1-1 |
| applicationCategory | enum | 申請区分（基礎/目的地/急速/超急速/経路）| 目的地充電 |
| constructionArea | enum | 施工エリア（東日本/中日本/西日本）| 東日本 |

#### 充電器情報
| フィールド | 型 | 説明 |
|-----------|-----|------|
| chargerCategory | string | 充電器種別（3kW/6kW/50kW/90kW/120kW/150kW/180kW） |
| chargerManufacturer | string | メーカー |
| chargerModel | string | 型番 |
| quantity | number | 台数 |

#### 担当者
| フィールド | 型 | 説明 |
|-----------|-----|------|
| salesDepartment | string | 営業担当部署 |
| caseManager | string | 案件担当 |
| sekoManager | string | Seko担 |
| constructionManager | string | 施工管理 |

#### 日程管理（20+フィールド）
- 発注日、着工前会議予定日/時間、安全書類提出日

- 着工Ready状態・確認日
- 現場担当者名・電話番号
- 産廃回収有無・荷姿・希望日時
- 実着工日・実完工日
- 完了報告書提出日・ステータス・承認日

### 3.3 充電器マスタ
7メーカー・20+モデルの管理:
- 日東工業: EVP-2G60J / EVPT-2G60J（通信あり）/ EVPTC2G60J
- 河村電器産業: EVコンポα ECMT3-6X-5 + OCPP制御ユニット
- ダイヘン: DQC050LD/M (50kW), DQC120 (120kW), DQC180 (180kW)
- ニチコン: NQC-TC504P (50kW), NQD-UCX04P (100kW)
- 新電元工業: SDQC2F50/90/90M/150
- NOEX: NXEV-0001 (TC 2024モデル)
- Zerova: AW32 (6kW/9.6kW)

### 3.4 図面データ
- 4種類: 設置場所見取図、平面図、配線ルート図、電気系統図
- ライフサイクル: 申請 → 変更 → 完成
- CADファイル: DWG, DXF, JWW, TFS対応
- 断面図: 3kW/6kW/急速 × 設置パターン（アンカー/基礎/壁面等）

### 3.5 新規現場調査依頼フロー（260411更新版）
1. テラ案件担当者 → KIZUKUアプリで案件化通知。トヨクモR8現場調査フォルダに詳細アップ
2. 弊社（西平）→ 案件種別・エリア・日程・設置案の詳細確認。協力業者選定
3. 弊社（西平）→ KIZUKUアプリ内で案件確認可能に設定変更
4. テラ営業/協力業者/弊社 → 契約者へ現場調査日程調整連絡
5. 協力業者 → 現場調査実施（契約者立会い有無確認済み）
6. 協力業者 → 現場調査報告書作成、KIZUKUにて写真等共有、テラへ提出

## 4. 外部ツール連携要件

### 4.1 Kizuku（工事管理アプリ）
- **現状**: 工事中の連絡、写真提出、安全書類提出、完了報告書作成
- **連携方針**: API連携 or データ同期（要調査）
- **主な機能**: トーク、報告書作成（承認依頼）、図書管理

### 4.2 Toyokumo（フォーム・ワークフロー）
- **現状**: 現地調査フォーム（R8 Nev現地調査）、進捗KPI、図面提出
- **連携方針**: kViewerでのデータ表示 + FormBridgeでの入力

### 4.3 Google Sheets（進捗共有シート）
- **現状**: 工事工程表シート（60+カラム、4時間毎更新）
- **連携方針**: 段階的に本プラットフォームへ移行
- **シート構成**: 進捗共有、工事工程表、R5Nev追加、R6Nev1回目/2回目、東京都他、電力申請、充電器手配確認、キャンセル案件、アスベスト管理

## 5. 非機能要件

### 5.1 パフォーマンス
- 案件数: 数百〜数千件/年度
- 同時アクセス: 50ユーザー程度

### 5.2 セキュリティ
- 認証: Supabase Auth
- 権限: テラ社員 / 協力会社（案件単位アクセス）
- 機密性: 補助金申請情報、契約金額は機密

### 5.3 可用性
- 工事現場からのモバイルアクセス対応
- オフライン時の写真一時保存

## 6. 業務ルール・制約

### 6.1 期限ルール
| ルール | 期限 |
|--------|------|
| 材料送付先入力 | 工事1ヶ月前 |
| 全資材確保 | 着工1週間前 |
| 安全書類提出 | 着工1週間前 |
| 着工前議事録提出 | 会議後2営業日以内 |
| 完了報告書提出 | 完工後3営業日以内 |
| 図面提出+承認依頼 | 毎月20日 |
| 承認完了 | 毎月25日 |
| 請求書送付 | 月初2営業日以内 |

### 6.2 工事変更ルール
- 補助金申請図面との変更発生 → 直ちにテラへ連絡
- 補助金上の変更可能確認 + お客様了承まで**工事中断**
- 変更図面提出（手書き可）

### 6.3 資材管理ルール
- 充電器は原則事務所配送（現着・2階以上不可）
- 保管中の盗難・破損は協力会社の賠償責任
- 参考価格: 普通充電器30万円、3kWコンセント10万円
- 工事期限の延長は補助金観点から**不可**
